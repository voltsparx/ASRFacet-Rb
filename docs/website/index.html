ASRFacet-Rb
MODULAR · EVENT-DRIVEN · OPERATOR-FOCUSED
A Ruby 3.2+ attack surface reconnaissance framework for authorized security testing. Combines passive discovery, recursive DNS and certificate enrichment, service mapping, HTTP fingerprinting, JavaScript endpoint mining, change tracking, correlation, and offline reporting in a single operator-focused workflow.

⚠ Authorized Use Only
Use ASRFacet-Rb only on systems you own or have explicit written permission to test. Built-in allow lists, exclusion lists, recon memory, and beginner-friendly explanations help operators stay inside authorized scope.
Core Capabilities
🔍
Passive Discovery
Aggregates subdomains from 8 passive intelligence sources — crt.sh, HackerTarget, Wayback, RapidDNS, AlienVault, Shodan, ThreatCrowd, BufferOver — before touching the target.
🌐
DNS & Cert Pivoting
Recursive DNS lookups and certificate SAN pivots validate and continuously expand the discovered asset set through feedback loops.
📡
Service Mapping
Threaded TCP port scanning with banner grabbing identifies exposed network services across all discovered IPs at configurable concurrency.
🕷️
Web Recon
HTTP fingerprinting, path probing, full crawl analysis, and JavaScript endpoint extraction map the complete web exposure surface.
📊
Scoring & Correlation
Asset scoring, findings generation, and cross-host correlation highlight the highest-priority targets — cutting through noise in large asset sets.
🔄
Change Tracking
Recon memory and intelligent diffing surface new, removed, or changed assets across repeated scans for monitoring workflows.
🧪
Safe Lab Mode
Built-in local validation lab lets you dry-run the entire pipeline against placeholder surfaces before touching any real authorized target.
🖥️
Web Control Panel
Local browser UI for saved sessions, live stage-by-stage activity, and full report browsing. Persists session state across crashes and reboots.
📖
Self-Documenting
Built-in explain, manual, wizard, and console mode teach operators recon methodology while the framework runs.
Getting Started
Installation
ASRFacet-Rb supports Bundler-based setup and cross-platform installers for Linux, macOS, and Windows.
Prerequisites
Ruby 3.2 or newer is required. Verify with ruby --version before proceeding.
Method 1 — Bundler (Development / Direct)
shell
# Clone the repository $ git clone https://github.com/voltsparx/ASRFacet-Rb.git $ cd ASRFacet-Rb # Install dependencies $ bundle install # Verify everything works (runs 49 tests + smoke checks) $ bundle exec rake # Confirm the CLI is functional $ bundle exec ruby bin/asrfacet-rb help $ bundle exec ruby bin/asrfacet-rb about $ bundle exec ruby bin/asrfacet-rb version
Method 2 — System Installers

The installers stage ASRFacet-Rb into its own application folder, create launchers on your user PATH, keep dependencies inside the install directory, and support a repo-local test mode. After installation, both asrfacet-rb and asrfrb are available as system commands.

Linux / macOS / Windows
# Linux $ bash install/linux.sh install $ bash install/linux.sh test $ bash install/linux.sh uninstall $ bash install/linux.sh update # macOS (also adds man page to MANPATH) $ bash install/macos.sh install # Windows (PowerShell) PS> .\install\windows.ps1 install PS> .\install\windows.ps1 test
Manual Page (Development)
shell
$ MANPATH="$PWD/man:$MANPATH" man asrfacet-rb

Recon Methodology
ConceptMeaningHow ASRFacet-Rb Applies It
Passive ReconUse third-party knowledge before touching the targetPassive runner aggregates known subdomains first, zero contact with target infrastructure
Active ReconValidate by making requests or opening connectionsDNS, ports, HTTP, crawl, and JS phases confirm real exposure on authorized targets
PivotingUse one discovery to find anotherDNS records, cert SANs, and knowledge graph edges feed new assets forward automatically
PrioritizationNot every asset deserves equal attentionAsset scoring and findings rank what matters most, filtering noise for the operator

Core Concepts
Scan Pipeline
Eight sequential stages transform a domain name into a fully correlated, scored attack surface map.
01
Passive Runner
Gathers known subdomains from all 8 external passive sources before any active probing begins. No traffic reaches the target during this stage.
→ Output: subdomains, passive errors
02
DNS + Certificate Discovery
Validates assets and pivots through DNS records and certificate Subject Alternative Names (SANs) to expand the target set. Graph edges connect related assets.
→ Output: DNS records, IPs, certificates, graph edges
03
Permutation + DNS Buster
Generates and actively validates candidate hostnames using bundled wordlists. Custom wordlists can be supplied with --wordlist.
→ Output: candidate subdomains, discovered subdomains
04
Discovery Feedback Loop
Feeds newly found hosts back into DNS and certificate analysis, iteratively expanding the asset set. This is the core of ASRFacet-Rb's event-driven architecture.
→ Output: expanded asset set
05
Port Engine
Identifies reachable services on all discovered IPs via threaded TCP scanning with banner grabbing. Thread count is configurable with --threads.
→ Output: open ports, banners, service nodes
06
HTTP + Crawl + JS + Correlation
Maps web exposure, path checks, scripts, JavaScript API endpoints, and cross-host correlation patterns. Generates asset scores and top-target rankings.
→ Output: HTTP responses, crawl data, JS endpoints, correlations, top assets
07
WHOIS + ASN
Adds ownership and infrastructure context to the knowledge graph. Connects domains and IPs to registrant data and autonomous system information.
→ Output: WHOIS records, ASN data
08
Vuln Hints + Monitoring
Generates findings and probabilistic hints, diffs results against recon memory, and persists scan state for future monitoring runs. Change tracking activates with --monitor.
→ Output: findings, change diff, probabilistic hints, memory updates

Passive-Only Usage
shell
# Basic passive footprint $ asrfacet-rb passive example.com # Passive with Shodan enrichment $ asrfacet-rb passive example.com --shodan-key YOUR_SHODAN_KEY # Passive output as JSON for downstream tools $ asrfacet-rb passive example.com --format json --output passive.json # Passive-only mode via the scan entrypoint $ asrfacet-rb scan example.com --passive-only

All global flags work with any command unless noted. Scan-specific flags apply only to scan and ports.

-f, --format FORMATcli, json, html, txtSelect the output renderer. Default is cli.
--top NintegerLimit top targets shown in CLI output. Keeps terminal focused.
Performance
FlagValuesDescription
-t, --threads Ninteger (default: 50)Set worker concurrency. Tune scan speed versus resource pressure.
--timeout SECsecondsSet the network timeout. Adapt to slow or unstable targets.
-v, --verboseflagPrint stage-by-stage progress to understand where time is spent.
Scope & Safety
FlagValuesDescription
--scope LISTcomma-separated domains/IPsAdd allowed domains or IPs. Keeps active probing inside the authorized boundary.
--exclude LISTcomma-separated domains/IPsBlock discovered assets from being touched. Avoids third-party or sensitive infrastructure.
--memoryflagSkip already confirmed subdomains. Reduces repeat work on repeated scans.
--monitorflagShow changes since the last recorded scan. Tracks drift between repeat recon runs.

Syntax Guide
Full syntax reference with port range formats, scope lists, output paths, and flag combinations.
General Syntax
syntax
asrfacet-rb COMMAND [TARGET] [FLAGS] [VALUES] # COMMAND — the action to perform (scan, passive, ports, dns, lab...) # TARGET — domain name or IP address (required for scan/passive/ports/dns) # FLAGS — modifiers starting with -- or - (see Global Flags) # VALUES — value for the preceding flag
Port Range Syntax
port formats
# Named sets asrfacet-rb scan example.com --ports top100 # 100 most common ports asrfacet-rb scan example.com --ports top1000 # 1000 most common ports # Numeric ranges asrfacet-rb ports host.com --ports 1-1000 # range asrfacet-rb ports host.com --ports 8000-9000 # custom range # Comma-separated lists asrfacet-rb ports host.com --ports 22,80,443,8443 # specific ports asrfacet-rb ports host.com --ports 80,443,3000,8080,8443
Scope & Exclusion Syntax
scope
# Allow specific domains only asrfacet-rb scan example.com --scope example.com,api.example.com # Exclude third-party or sensitive infrastructure asrfacet-rb scan example.com --exclude dev.example.com,staging.example.com # Combine scope and exclusions asrfacet-rb scan example.com \ --scope example.com,api.example.com \ --exclude dev.example.com
Output Syntax
output
# HTML report saved to disk asrfacet-rb scan example.com --format html --output report.html # JSON for downstream processing asrfacet-rb scan example.com --format json --output results.json # Plain text export asrfacet-rb scan example.com --format txt --output notes.txt # Terminal output only (default) asrfacet-rb scan example.com --format cli
Combined Flag Syntax
combined
# Deep scan with all options asrfacet-rb scan example.com \ --ports top1000 \ --threads 100 \ --timeout 15 \ --scope example.com \ --monitor \ --memory \ --format html \ --output deep-report.html \ --verbose # Alias short-form asrfacet-rb s example.com -p top1000 -t 75 -f html -o out.html

Web Session Mode
A local-only browser control panel that gives you a GUI for configuring scans, watching live activity, and browsing reports.
launch web session
# Default — binds to 127.0.0.1:4567 asrfacet-rb --web-session # Custom host and port asrfacet-rb --web-session --web-host 127.0.0.1 --web-port 8080 # Via command alias asrfacet-rb web asrfacet-rb w

Session drafts are persisted under ~/.asrfacet_rb/web_sessions/, so configuration survives accidental browser closes, process crashes, and power loss. Running sessions use heartbeats so stale runs are marked interrupted without clobbering a freshly active session.

The dashboard includes an About section and an embedded documentation viewer backed by the repository docs/ content. Reports saved by any scan run (CLI, JSON, HTML, TXT) are all browsable from within the web panel.

Reference
Core Modules
The internal architecture is built around thread-safe, event-driven modules with strict execution ownership.
ASRFacet::Core::Target
Normalizes the target domain and caches HTTP fetches
Provides a stable base object for the entire run
ASRFacet::ResultStore
Thread-safe result collection
Keeps discoveries deduplicated and serializable across all engines
ASRFacet::EventBus
Internal event routing between engines
Lets discovered assets feed later processing stages automatically
ASRFacet::Core::KnowledgeGraph
Relationship graph for domains, IPs, services, ASNs, and findings
Supports pivots like domain-to-IP and IP-to-service for correlation
ASRFacet::Core::ReconMemory
Persistent per-target scan history
Enables monitoring and skip-known behavior across repeated runs
ASRFacet::Core::ScopeEngine
Enforces allow and exclude lists at runtime
Helps prevent out-of-scope probing on authorized engagements
ASRFacet::Core::NoiseFilter
Filters low-value HTTP output and duplicate findings
Keeps reports focused on meaningful results
ASRFacet::Core::CircuitBreaker
Isolates repeatedly failing components
Improves resiliency during unstable or slow network scans
Architecture Note
Execution ownership stays intentionally strict: the scheduler owns orchestration, execution helpers run work, and engines do not create their own competing control loops.
